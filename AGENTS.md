# Reports Browser Agent Instructions

This project is a React/Vite app for uploading, storing, and browsing Kampfinsel reports through Supabase.

## Source Boundaries

- `src/App.tsx` owns page-level state and application wiring.
- `src/domain/` contains report parsing, report models, filtering, and other business logic.
- `src/lib/` contains integration helpers such as Supabase access.
- `supabase/` contains database migrations and schema-facing assets.
- `dist/` is generated by Vite and should not be edited by hand.

## Security

- Only use the Supabase anon key in browser code.
- Never put a Supabase service-role key into this app, userscripts, the loss calculator, commits, docs, or examples.
- Keep `.env.local` local-only. Prefer `.env.example` when documenting required variables.

## Verification

For source changes, run:

```bash
npm run typecheck
npm test
npm run build
```

Use `npm run dev -- --port 5174 --strictPort` for local browser checks when UI behavior changes.

## Coding Guidelines

- Keep React components focused on rendering and interaction.
- Prefer small pure functions in `src/domain/` for parsing, normalization, and filtering.
- Keep Supabase queries and environment handling isolated in `src/lib/`.
- Preserve existing migrations unless intentionally creating a new migration.
